Great question! This is actually a **usability feature** in Claude Code's skill system.

## How Skill Invocation Works:

### Short Form (Automatic Resolution)
- When you type `/setup`, Claude Code automatically resolves it to `sdlc:setup`
- You can use the short form **as long as there's no ambiguity**
- If multiple plugins have a `setup` skill, you'd need to use the full namespace to disambiguate

### When Namespaces Matter:
The namespace prefix (`sdlc:`) serves these purposes:

1. **Organization**: Groups related skills together in the plugin structure
2. **Conflict Prevention**: Allows multiple plugins to have skills with the same name
3. **Explicit Invocation**: You can always use `/sdlc:setup` if you want to be explicit

### Example Scenario:
```
Plugin A provides: editor:format
Plugin B provides: linter:format

# These would require namespaces:
/editor:format   (explicit)
/linter:format   (explicit)

But if only one plugin has "format":
/format          (auto-resolves to the only available one)
```

## In Your Case:

Since your `sdlc` plugin is likely the only one providing skills like `setup`, `work`, `pr`, etc., Claude Code lets you use the convenient short form. The namespace is there for the plugin system's organization and to prevent future conflicts, but doesn't burden users with extra typing when it's unnecessary.

**TL;DR**: You can use `/setup`, and Claude will find `sdlc:setup` automatically. Use `/sdlc:setup` only when you need to be explicit or if there are naming conflicts.
