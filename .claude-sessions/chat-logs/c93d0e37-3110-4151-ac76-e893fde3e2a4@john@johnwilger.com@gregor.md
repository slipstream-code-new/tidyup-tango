**Command module structure (using commanded_boilerplate):**
```elixir
defmodule TodoList.Task.Commands.AddTask do
  use Commanded.Boilerplate.Command, identifier: :task_id

  inputs do
    field :task_id, TodoList.TaskId.Ectotype
    field :description, TodoList.TaskDescription.Ectotype
  end

  aggregate do
    field :task_count, :integer, default: 0

    def apply(state, %TodoList.Task.Events.TaskAdded{}) do
      %{state | task_count: state.task_count + 1}
    end
  end

  @impl true
  def handle(aggregate, command) do
    # Domain logic here. Return {:ok, [events]} or {:error, %DomainErrorType{}}
    position = aggregate.task_count + 1
    {:ok, [%TodoList.Task.Events.TaskAdded{
      task_id: command.task_id,
      description: command.description,
      position: position
    }]}
  end

  @impl true
  def authorize(_aggregate, _command), do: :ok

  register_command(TodoList.Router, TodoList.App)
end
```

Key aspects of `commanded_boilerplate`:
- `use Commanded.Boilerplate.Command, identifier: :task_id` — sets up routing, validation middleware, authorization middleware
- `inputs do ... end` — defines command input fields as typed struct
- `aggregate do ... end` — inline aggregate with only fields needed for THIS command's decisions
- `handle/2` — command handler receiving aggregate state + command, returns events or error
- `authorize/2` — returns `:ok` unconditionally (single-user app, no auth)
- `register_command/2` — registers with the Commanded router

### Context Module: TodoList.Tasks

defmodule TodoList.Tasks do
  @moduledoc "Context module for task management."
  
  alias TodoList.App
  alias TodoList.Task.Commands.AddTask

  @spec add_task(String.t()) :: {:ok, String.t()} | {:error, term()}
  def add_task(description) do
    task_id = UUIDv7.generate()
    command = %AddTask{task_id: task_id, description: description}

      :ok -> {:ok, task_id}
      {:error, reason} -> {:error, reason}
    end

  @spec active_tasks() :: [map()]
  def active_tasks do
    import Ecto.Query
    TodoList.Repo.all(
      from t in TodoList.Task.Projections.TaskProjection,
        where: t.completed == false and t.deleted == false,
        order_by: [asc: t.position]
    )
  end
end

Since each command gets its own aggregate stream, there's no way for AddTask's aggregate to know about other tasks. **The position should be determined by the context module**, not the aggregate. The context can query the current count of tasks and pass position as a field on the command.

The aggregate cannot track a global count. **Solution**: The context module (`TodoList.Tasks.add_task/1`) should:

  2. Set position = max_position + 1
