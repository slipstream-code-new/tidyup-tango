### What This Does

**When:** `git.worktrees: true` in config

**Invoke:** `/sdlc:work`

## When to Use

**Use this skill when:**
- Starting work on a new task
- Returning to work after a break
- Switching between tasks
- User asks "what should I work on next?"

**Don't use when:**
- No tasks exist yet (use `/sdlc:plan` to create tasks first)
- Need to create PR (use `/sdlc:pr` after work complete)
- Addressing PR feedback (use `/sdlc:review` instead)
- Project not set up (run `/sdlc:setup` first)

**Related skills:**
- `/sdlc:plan` - Create tasks from event model
- `/sdlc:pr` - Create pull request
- `/sdlc:review` - Address PR feedback
- `/sdlc:complete` - Mark task done

## Acceptance Criteria: Foundation Slices (P0)

### 📋 Slice 1: Add Task

**User Story:**  
*As a user, I want to add a new task with a description, so that I can capture things I need to do.*

**Command Validation:**
- Description: 1-500 characters, trimmed, non-empty
- Auto-generates UUID task ID
- Auto-assigns priority at bottom of list (max + 1, or 0 if empty)
- New tasks start incomplete

**GWT Scenarios:**

1. **Add first task to empty list**
   - Given: (no prior events)
   - When: AddTask { description: "Buy groceries for the week" }

2. **Add task to existing list**

   - Then: TaskAdded { priority: 2 }

### 👁️ Slice 2: Task List View

**User Story:**  
*As a user, I want to see all my tasks in priority order, so that I know what to work on next.*

**Projection Logic:**
- Projects all task events (Added, Completed, Incomplete, Priority Changed, Description Changed, Deleted)
- Displays sorted by priority (ascending)
- Shows completion status, description, timestamps

**GWT Scenarios:**

1. **Display newly added task**
   - Given: TaskList { tasks: [] }
   - When: TaskAdded event
   - Then: Task appears with isComplete: false, proper timestamps

2. **Update when marked complete**
   - Given: Task exists (incomplete)
   - When: TaskMarkedComplete event
   - Then: isComplete: true, completedAt set

3. **Reflect priority changes**
   - Given: 3 tasks (priorities 0, 1, 2)
   - When: TaskPriorityChanged (task 2 → priority 0)
   - Then: List re-sorts, other priorities shift

4. **Update description after edit**
   - When: TaskDescriptionChanged event
   - Then: Description updated, lastModifiedAt set

5. **Remove task after deletion**
   - When: TaskDeleted event
   - Then: Task removed from list

6. **Mark task incomplete**
   - When: TaskMarkedIncomplete event
   - Then: isComplete: false, completedAt: null
